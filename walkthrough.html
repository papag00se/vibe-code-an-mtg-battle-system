Step 0 — Before you prompt

The words you need first

If you have never used an AI coding agent, learn this vocabulary before running anything. Full detail lives in docs/02_codex_and_ai_foundations.md.

LLM

A large language model — the engine that predicts useful next text. It powers everything Codex does.

Inference

One run of that engine: text in, text out. Every time Codex "thinks," it runs inference.

Prompt

Your instructions. Specific beats vague. Prompts are your main lever of control.

Agent

An LLM that can also act — read files, edit code, run commands — not just chat.

Codex

The coding agent you steer here. It reads your repo and proposes changes in a loop.

Context

Everything the model can currently see: your prompt, the files it read, and the recent conversation.

How you work

The vibe-coding loop

You drive in short turns. Small request, review, steer again. You are the senior engineer; Codex is the fast junior.

Give one small, specific goal.
Provide context and constraints.
Let Codex implement.
Inspect the diff — read what changed.
Run the app and the tests yourself.
Ask Codex to explain its change.
Request the next small improvement. Repeat.

Guardrails

Permission levels (including "YOLO")

An agent that can run commands needs limits. Names vary by version, but they fall into three tiers. For this tutorial, start strict.

Read-only / suggest start here

Codex proposes edits and commands, but you approve each action. Best for learning and unfamiliar code.

Auto-edit with care

Codex edits files on its own but asks before running commands. Fine once you trust the task.

Full-auto "YOLO" danger

No approval prompts — reads, edits, and runs commands freely. Only on committed, reversible, sandboxed work.

Rule of thumb:
If you would not let a brand-new intern run that command
unsupervised on your laptop, do not let an agent do it
in full-auto either.
